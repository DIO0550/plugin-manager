# Hooks 自動変換 - ラッパースクリプト生成仕様

> **機能**: [Hooks 自動変換](./index.md)
> **ステータス**: 下書き

## 概要

Claude Code 形式のフックスクリプトを Copilot CLI 環境で動作させるためのラッパースクリプトを生成する。Copilot CLI から渡される stdin を Claude Code スキーマに変換し、元のスクリプトを呼び出し、Claude Code スキーマの stdout を Copilot CLI スキーマに逆変換する。

## ラッパースクリプトの構造

```
Copilot CLI
  │ stdin (Copilot CLI スキーマ)
  ▼
┌─────────────────────────┐
│   ラッパースクリプト      │
│  1. stdin を変換         │
│  2. matcher フィルタ     │
│  3. 元スクリプトを実行    │
│  4. stdout を逆変換      │
│  5. exit code を変換     │
└─────────────────────────┘
  │ stdout (Copilot CLI スキーマ)
  ▼
Copilot CLI
```

## ビジネスロジック

### BL-001: stdin 変換（Copilot CLI → Claude Code）

Copilot CLI から受け取った stdin JSON を Claude Code スキーマに変換して元スクリプトに渡す。

#### PreToolUse / PostToolUse の場合

| 変換 | 入力 (Copilot CLI) | 出力 (Claude Code) |
|------|-------------------|-------------------|
| タイムスタンプ除去 | `"timestamp": 1704614600000` | — |
| セッションID追加 | — | `"session_id": "plm-bridge"` |
| CWD | `"cwd": "/project"` | `"cwd": "/project"` |
| ツール名変換 | `"toolName": "bash"` | `"tool_name": "Bash"` |
| ツール入力変換 | `"toolArgs": "{\"command\":\"...\"}"` | `"tool_input": {"command": "..."}` |
| ツール結果変換 (PostToolUse) | `"toolResult": {"resultType":"success","textResultForLlm":"..."}` | `"tool_response": {"resultType":"success","textResultForLlm":"..."}` |

#### SessionStart の場合

| 変換 | 入力 (Copilot CLI) | 出力 (Claude Code) |
|------|-------------------|-------------------|
| source マッピング | `"new"` | `"startup"` |
| source マッピング | `"resume"` | `"resume"` |
| source マッピング | `"startup"` | `"startup"` |
| initialPrompt | `"initialPrompt": "..."` | — (除去) |

#### UserPromptSubmit の場合

| 変換 | 入力 (Copilot CLI) | 出力 (Claude Code) |
|------|-------------------|-------------------|
| prompt | `"prompt": "..."` | `"prompt": "..."` |

構造は類似のため、`timestamp` 除去 + `session_id` 追加のみ。

### BL-002: ツール名変換（Copilot CLI → Claude Code）

| Copilot CLI (`toolName`) | Claude Code (`tool_name`) |
|--------------------------|--------------------------|
| `bash` | `Bash` |
| `view` | `Read` |
| `create` | `Write` |
| `edit` | `Edit` |
| `glob` | `Glob` |
| `grep` | `Grep` |
| `web_fetch` | `WebFetch` |
| `task` | `Agent` |
| `ask_user` | — (変換なし、そのまま渡す) |
| `memory` | — (変換なし、そのまま渡す) |
| `powershell` | `Bash` (近似マッピング) |

未知のツール名はそのまま渡す（フォワードコンパティビリティ）。

### BL-003: matcher フィルタリング

元の Claude Code 設定で `matcher` が指定されていた場合、ラッパースクリプト内でフィルタリングを実行する。

```bash
# matcher が "Bash" の場合
TOOL_NAME=$(echo "$CLAUDE_INPUT" | jq -r '.tool_name')
if [[ ! "$TOOL_NAME" =~ ^Bash$ ]]; then
  exit 0  # マッチしない場合は何もせず許可
fi
```

- matcher は正規表現として解釈する
- matcher が空文字列または未設定の場合はフィルタなし（全マッチ）
- ツール名の比較は変換後の Claude Code 形式（PascalCase）で行う

### BL-004: stdout 変換（Claude Code → Copilot CLI）

元スクリプトの stdout を Copilot CLI が期待するスキーマに変換する。

#### PreToolUse の場合

```
入力（元スクリプトの stdout）:
{
  "hookSpecificOutput": {
    "hookEventName": "PreToolUse",
    "permissionDecision": "deny",
    "permissionDecisionReason": "Dangerous command"
  }
}

出力（Copilot CLI への stdout）:
{
  "permissionDecision": "deny",
  "permissionDecisionReason": "Dangerous command"
}
```

変換ルール:
- `hookSpecificOutput` をアンラップする
- `hookEventName` を除去する
- `updatedInput` / `additionalContext` は除去（Copilot CLI に対応なし）
- `continue` / `stopReason` / `suppressOutput` / `systemMessage` は除去

#### PreToolUse 以外の場合

Copilot CLI では `preToolUse` 以外のイベントで stdout は無視されるため、元スクリプトの出力をそのまま破棄する。

### BL-005: exit code 変換

| 元スクリプトの exit code | ラッパーの出力 | Copilot CLI での意味 |
|------------------------|--------------|---------------------|
| `0` | exit `0` + stdout を変換して出力 | 成功（stdout をパース） |
| `2`（ブロック） | exit `0` + `{"permissionDecision":"deny","permissionDecisionReason":"<stderr>"}` | deny として処理 |
| `1` / その他 | exit `0` + 出力なし | 成功扱い（元スクリプトのエラーを無視） |

**重要:** Copilot CLI では非ゼロ exit code はエラー扱い（ログしてスキップ）であり、ブロック手段にならない。Claude Code の exit 2（ブロック）を実現するには、exit 0 + JSON で `permissionDecision: "deny"` を返す必要がある。

### BL-006: 環境変数のブリッジ

ラッパースクリプトが元スクリプトに Claude Code 互換の環境変数を提供する。

| 環境変数 | 値のソース |
|---------|----------|
| `CLAUDE_PROJECT_DIR` | Copilot CLI の stdin から `cwd` を取得 |
| `CLAUDE_PLUGIN_ROOT` | PLM が配置時に埋め込む（プラグインのルートパス） |

`CLAUDE_FILE_PATHS` と `CLAUDE_ENV_FILE` は Copilot CLI から取得できないため、設定しない。

## 生成されるラッパースクリプトの例

### command フックのラッパー

```bash
#!/bin/bash
# Generated by PLM - Claude Code to Copilot CLI hook wrapper
# Original: ./validate-bash.sh
# Matcher: Bash
set -euo pipefail

# --- stdin 変換 (Copilot CLI → Claude Code) ---
COPILOT_INPUT=$(cat)
CLAUDE_INPUT=$(echo "$COPILOT_INPUT" | jq '{
  session_id: "plm-bridge",
  cwd: .cwd,
  tool_name: (
    {bash:"Bash",view:"Read",create:"Write",edit:"Edit",
     glob:"Glob",grep:"Grep",web_fetch:"WebFetch",task:"Agent",
     powershell:"Bash"}[.toolName] // .toolName
  ),
  tool_input: (.toolArgs | fromjson)
}')

# --- matcher フィルタ ---
TOOL_NAME=$(echo "$CLAUDE_INPUT" | jq -r '.tool_name')
if [[ ! "$TOOL_NAME" =~ ^Bash$ ]]; then
  exit 0
fi

# --- 環境変数ブリッジ ---
export CLAUDE_PROJECT_DIR=$(echo "$COPILOT_INPUT" | jq -r '.cwd')
export CLAUDE_PLUGIN_ROOT="@@PLUGIN_ROOT@@"

# --- 元スクリプト実行 ---
RESULT=$(echo "$CLAUDE_INPUT" | "$CLAUDE_PLUGIN_ROOT/hooks/validate-bash.sh" 2>/tmp/plm-hook-stderr-$$ || true)
EXIT_CODE=${PIPESTATUS[1]:-$?}
STDERR=$(cat /tmp/plm-hook-stderr-$$ 2>/dev/null || echo "")
rm -f /tmp/plm-hook-stderr-$$

# --- exit code + stdout 変換 (Claude Code → Copilot CLI) ---
if [ "$EXIT_CODE" -eq 0 ] && [ -n "$RESULT" ]; then
  # 成功: hookSpecificOutput をアンラップ
  echo "$RESULT" | jq '
    if .hookSpecificOutput then
      .hookSpecificOutput | del(.hookEventName)
    else
      .
    end
  '
elif [ "$EXIT_CODE" -eq 2 ]; then
  # ブロック: deny レスポンスを生成
  REASON=${STDERR:-"Blocked by hook"}
  jq -n --arg reason "$REASON" '{
    permissionDecision: "deny",
    permissionDecisionReason: $reason
  }'
fi
# exit 1 / その他: 出力なし（Copilot CLI は許可扱い）
```

### http フックのラッパー

```bash
#!/bin/bash
# Generated by PLM - Claude Code HTTP hook wrapper
# URL: http://localhost:8080/webhook
set -euo pipefail

INPUT=$(cat)

RESPONSE=$(echo "$INPUT" | curl -s -w "\n%{http_code}" -X POST \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d @- \
  "http://localhost:8080/webhook")

HTTP_CODE=$(echo "$RESPONSE" | tail -1)
BODY=$(echo "$RESPONSE" | sed '$d')

if [ "$HTTP_CODE" -ge 200 ] && [ "$HTTP_CODE" -lt 300 ] && [ -n "$BODY" ]; then
  echo "$BODY" | jq '
    if .hookSpecificOutput then
      .hookSpecificOutput | del(.hookEventName)
    else
      .
    end
  '
fi
```

## ファイル配置

ラッパースクリプトは Hooks 配置先ディレクトリ内の `wrappers/` サブディレクトリに生成する。

```
.github/hooks/{marketplace}/{plugin}/
├── hooks.json              # 変換済み Copilot CLI 形式の設定
└── wrappers/
    ├── validate-bash.sh    # command フックのラッパー
    └── http-webhook.sh     # http フックのラッパー
```

- ラッパースクリプトには実行権限 (`chmod +x`) を付与する
- `@@PLUGIN_ROOT@@` プレースホルダーは配置時にプラグインの実際のパスに置換する

## エラーハンドリング

| エラーケース | 発生条件 | 振る舞い |
|:------------|:---------|:---------|
| jq が未インストール | ラッパー実行時に jq が見つからない | exit 0（許可扱い、stderr に警告） |
| 元スクリプトが見つからない | パスが不正または削除済み | exit 0（許可扱い、stderr に警告） |
| 元スクリプトの stdout が不正 JSON | JSON パースエラー | exit 0（許可扱い、元の出力を無視） |
| toolArgs が不正 JSON | `fromjson` 失敗 | 空オブジェクト `{}` を `tool_input` に設定 |

**原則:** ラッパースクリプトのエラーは元のエージェント実行をブロックしない（フェイルオープン）。

## 制限事項

- `jq` がシステムにインストールされている必要がある
- Windows (`powershell`) 向けのラッパーは生成しない
- Claude Code の `CLAUDE_ENV_FILE`（環境変数永続化）は Copilot CLI で再現できない
- `prompt` / `agent` フックはスタブスクリプト（常に許可）を生成する
